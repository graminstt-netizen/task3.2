3.2 Реализовать арифметику больших чисел с помощью битовых массивов.

Битовый массив реализуется через массив слов целочисленного беззнакового типа задаваемого при компиляции: 
должна быть возможность указать любой поддерживаемый компилятором тип (unsigned char, unsigned short, unsigned int, unsigned long, 
unsigned long long, size_t, uint8_t, uint16_t, uint32_t, uint64_t, uintptr_t). В битовом массиве будут хранится элементы фиксированной битовой
длины N (задаемой при компиляции). Интерфейс массива предоставляет функции чтения/записи элементов по индексам. Все элементы хранятся в памяти 
компактно: в одном слове массива могут хранится биты разных элементов массива. Например, если слово будет типа unsigned int, а размер N=2, то в 
каждом слове будут биты шестнадцати элементов. Длина N возможна от 2 до 64 чтобы элемент мог поместиться в unsigned long long (для расположения 
в переменных, передачи и возврата из функций).

Большое число представляется в виде N-битовых кусочков и операции выполняются над этими кусочками 
(доступ к ним осуществляется через функции чтения/записи битового массива).

Условия для оценки 3

Возможные значения N {4, 8, 16, 32, 64}.

Реализовать следующие операции:
- сложение и вычитание больших чисел;
- вывод большого числа;
- инициализация большого числа из строки шестнадцатеричных (десятичных по желанию) символов.


Условия для оценки 4

N может быть любым от 2 до 64.

Дополнительно реализовать операции умножения и деления больших чисел.
Для хранения отрицательных чисел использовать дополнительный код (на уровне всего массива), но выводить и принимать строку в обычном виде (с символом минуса).
